## Findings Considered And Rejected

- **Production dependency vulnerabilities — status changed; now Plan 005.** The
  first pass recorded `npm audit ... found 0 vulnerabilities` and rejected dep
  work. At the same commit, `npm audit` now reports **3 moderate, 0 high/critical**
  (advisory DBs are time-dependent, so the first pass was honest). All three are
  **dev/build-time only** (`postcss` via Tailwind/autoprefixer, `js-yaml` via
  eslint, `next` inheriting `postcss`). Promoted to Plan 005 as P3 hygiene.
- **Leader-dashboard performance micro-optimizations** (repeated `filter()` vs
  Map memoization, per-group N+1, unbounded membership fetch in
  `lib/dashboard/queries.ts`): rejected as standalone perf work because the code
  path is **dormant — no live caller**. The right move is deletion, not
  optimization; folded into Plan 004.
- **Missing `default:`/exhaustiveness guard in `transitionsFor()`**
  (`components/admin/follow-ups/follow-up-status-controls.tsx`): not worth a plan.
  `tsconfig` has `strict: true` and the function has an annotated `: Action[]`
  return, so a future non-exhaustive `FollowUpStatus` would fail `tsc` (the CI
  typecheck gate) rather than return `undefined` at runtime. Defensive nicety
  only.
- **Care-note read-model column-pinning tests**: not separately planned. The
  privacy-critical reads are already covered by
  `lib/supabase/__tests__/sc4-private-note-read-models.test.ts` and
  `read-model-columns.test.ts`; the audit's "14 untested read modules" framing
  over-stated the gap. (Plan 006 covers the genuinely untested _write_-action
  files, which is the real coverage hole.)
- **Runtime service-role / direct-write / select-star regressions**: rejected
  again this pass. A dedicated security re-audit found zero invariant violations;
  the `tests/fitness/**` suite (no-service-role, no-select-star,
  no-direct-table-writes, no-hardcoded-identity, write-rpc-audit-pairing,
  leader-allowlist-no-admin-private, …) is green and gates CI.
- **Tracked local secret cleanup**: rejected because
  `supabase/.env.test-auth.local` is ignored by `.gitignore` and is not tracked by
  git.
- **Frozen pre-pivot surfaces' design-system debt** (master calendar, guests,
  planning, …): not a finding. It is intentional and documented (ADR 0024 — pay
  the migration cost on revival). Listed here so it isn't re-audited as accidental
